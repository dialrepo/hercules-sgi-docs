# Hércules : REP\-CSP\-0022 \- Listado general de solicitudes \- Formato rtf







### Diseño Informe

Se debe de generar un informe en formato RTF compatible con DOCX que reciba como parámetros de entrada el título del informe y la lista de bloques de datos a incluir. El informe contendrá la información detallada de solicitudes. Se listarán todas las solicitudes recuperadas con el detalle de las mismas.  Se incluirá el parámetro de entrada "título" como título del informe.

El listado se mostrará agrupado por solicitud. Cada vez que se cambie de solicitud se resaltará la línea de texto en el informe y a continuación comenzará a listarse su detalle.

El detalle se visualizará en función del bloque de datos:

* A dos columnas. Siendo la primera columna la etiqueta de la misma y la segunda columna el contenido recuperado con los datos de la solicitud.
* En varias filas, pero también a dos columnas, de forma que varias filas compartan una misma columna de título. La primera columna será el título y en la segunda columna, saltando de fila, se mostrará el contenido.  ejemplo ![](/attachments/597853192/597884748.png)

Habrá bloques de datos que se repitan, en este caso la etiqueta será representativa del bloque de datos, e incluirá entre paréntesis el nombre de las columnas que se mostrarán para cada elemento. Cada elemento repetido se mostrará de alguna de las formas indicadas previamente (dependiendo de cada bloque de datos).

A continuación se indica el mapeo de los nombres de columnas del fichero de salida con la tabla y campo del SGI desde el que deben ser recuperadas. Se indica también el bloque de datos al que pertenece cada columna, así como el modo en el que se visualizará (a dos columnas o repeticiones de filas).

  


Ejemplo de formato de datos:  [proyectos\_with\_list.rtf](https://confluence.um.es/confluence/download/attachments/121673631/proyectos_with_list.rtf?version=1&modificationDate=1638171055557&api=v2 "https://confluence.um.es/confluence/download/attachments/121673631/proyectos_with_list.rtf?version=1&modificationDate=1638171055557&api=v2") al que se de debe de incorporar la visualización en varias filas, de manera semejante a la imagen mostrada previamente.
