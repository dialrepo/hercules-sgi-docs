# Hércules : IU\-CSP\-0403\-002 \- Búsqueda ampliada y listado de proyectos \- Investigador



## Formulario Búsqueda ampliada y listado de proyectos \- Investigador

Pantalla que muestra el formulario de búsqueda ampliada, que permite aplicar filtros sobre el listado de proyectos para los que ACT\-CSP\-001\-Investigador figura en el equipo. 



|  | | | |
| --- | --- | --- | --- |
| Nombre | | Tipo | Características / Notas |
| Formulario de búsqueda de proyectos. De manera implícita esta búsqueda llevará establecido un filtro que limite los proyectos a los activos (campo "activo" a "true"), un filtro por el estado actual del proyecto, de forma que ACT\-CSP\-001\-Investigador no tendrá visibilidad de los proyectos cuyo estado actual sea "Borrador", y un filtro que establezca que ACT\-CSP\-001\-Investigador deba figurar en el equipo de proyecto. Si se debe aplicar algún filtro adicional sobre la fecha de pertenencia al equipo deberá estar marcado el check "proyectos con participación actual". | | | |
| Proyectos con participación actual | | CheckOpcional | Si este check está marcado llevará implícito:* Que se aplique implícitamente el filtro de búsqueda que marque que ACT\-CSP\-001\-Investigador deba de pertenecer al equipo de proyecto en la fecha actual, es decir, que ACT\-CSP\-001\-Investigador figure en al tabla "proyecto equipo" con un rango "fecha de inicio" y "fecha de fin" que contenga la fecha actual ("fecha inicio" \<\= fecha actual \<\= "fecha fin").  No será obligatorio que se marque este check, conllevando en este caso que no se apliquen filtros sobre el momento de pertenencia de ACT\-CSP\-001\-Investigador al equipo de proyecto, pero sí será imprescindible que figure en el equipo (con unas fechas de participación u otras).No estará seleccionado por defecto. |

| Identificador interno | | NuméricoOpcional | Código numérico generado automáticamente por el SGI al crear el proyecto.Filtro a aplicar sobre el campo "ID" de la tabla "proyecto". |
| Código identificación SGE | | TextoOpcional | Código de identificación del proyecto económico del SGE con el que se vincula el proyecto SGI. Podrá ser alfanumérico.Filtro a aplicar sobre el campo "proyecto SGE Ref" de la tabla "proyecto proyectoSGE". |
| Referencia en entidad convocante | | TextoOpcional | Cadena de texto para indicar la referencia que ha otorgado al proyecto la entidad convocante.Filtro a aplicar sobre el campo "código externo" de la tabla "proyecto" |
| Fecha inicio \- entre \- y \- | | FechaOpcional | Rango de fechas a aplicar sobre el campo "fecha inicio" de la tabla "proyecto". Será obligatorio que el rango de búsqueda se establezca con los dos límites ("entre" e "y") informados. Si se rellena el campo "fecha de inicio entre", se rellenará automáticamente el rango final (campo "fecha de inicio y") con la fecha actual. |
| Fecha fin / Fecha fin definitiva \- entre \- y \- | | FechaOpcional | El filtro aplicará indistintamente sobre la fecha de fin inicial y la fecha de fin definitiva del proyecto.Rango de fechas a aplicar sobre los campos "fecha fin" y "fecha fin definitiva" de la tabla "proyecto". Será suficiente con que al menos una de las dos fechas cumpla el filtro. Será obligatorio que el rango de búsqueda se establezca con los dos límites ("entre" e "y") informados. Si se rellena el campo "fecha de fin entre", se rellenará automáticamente el rango final (campo "fecha de fin y") con la fecha actual. |
| Estado | | SelectorOpcional | Selector cargado con solo dos valores: Concedido, Renunciado y Rescindido.Aunque el listado de estados posibles de un proyecto sea: Borrador, Concedido,Renunciado y  Rescindido, ACT\-CSP\-001\-Investigador solo tendrá acceso a los proyectos que se hayan consolidado, es decir, aquellos cuyo estado actual no sea "Borrador" |
| Listado de proyectos. La tabla de resultados se mostrada ordenada por defecto por ID interno, en orden de mayor a menor. | | | |
| ID interno | | Numérico entero | Código numérico generado automáticamente por el SGI para identificar de forma única al proyecto.Se corresponde con el campo "ID" de la tabla "proyecto" |
| Código SGE | | Texto | Código de identificación del proyecto económico del SGE con el que se vincula el proyecto SGI.Se corresponde con  el campo "proyecto SGE Ref" de la tabla "proyecto proyectoSGE". |
| Título | | Texto | Título del proyecto.Se corresponde con el campo "título" de la tabla "proyecto". |
| Acrónimo | | Texto corto | Código de referencia del proyecto.Se corresponde con el campo "acrónimo" de la tabla "proyecto". |
| Referencia en entidad convocante | | Texto | Referencia que la entidad convocante asigna al proyecto.Se corresponde con el campo "código externo" de la tabla "proyecto". |
| Fecha inicio | | Fecha | Fecha de inicio del proyecto.se corresponde con el campo "fecha inicio" de la tabla "proyecto". |
| Fecha fin | | Fecha | Fecha de fin inicial del proyecto.Se corresponde con el campo "fecha fin" de la tabla "proyecto". |
| Fecha fin definitiva | | Fecha | Fecha de fin definitiva del proyecto. Se corresponde con el campo  "fecha fin definitiva" de la tabla "proyecto". |
| Finalizado | | Boolean | Indicará si el proyecto se encuentra finalizado. Se mostrará con los valores "sí" o "no".Para obtener el valor a mostrar se comprobará:* Que la fecha actual sea mayor que la fecha fin inicial (campo "fecha fin") en el caso de que no esté informada la fecha fin definitiva (campo "fecha fin definitiva"). * Que la fecha actual sea mayor que la fecha fin definitiva (campo "fecha fin definitiva") en el caso de que este campo esté informado. |
| Prorrogado | | Boolean | Indicará si el proyecto tiene alguna prórroga. Se mostrará con los valores "sí" o "no".Para obtener el valor a mostrar se comprobará si existe algún registro en la tabla "proyecto prórroga" asociado al proyecto. |
| Estado proyecto | | Texto corto | Estado del proyecto.Nombre recuperado de la tabla "estado proyecto", a partir del campo "estado" de la tabla "proyecto". |
